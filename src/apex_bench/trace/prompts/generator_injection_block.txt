## Strategy cheatsheet from prior cases in this area

Below is a cheatsheet of ATOMIC STRATEGY BULLETS curated from prior
cases on structurally similar problems. Each bullet captures a
procedure, framework, convention, definitional pitfall, or output-
format requirement worth knowing for this kind of case. Bullets carry
usage counters: `helpful` is how often the bullet was cited on a
case that was judged correct, `harmful` is how often it was cited on
an incorrect case, `usage` is the total cite count.

Use this as procedural memory, not as evidence and not as an answer
key. First solve the current case from its actual prompt, attachments,
and source documents. Consult a bullet only when its trigger matches
the current task structure, then adapt the method to the current data.

Do NOT copy specific numerical values, named entities, file paths, or
named references from a bullet — values in bullets are illustrative;
the current case has different actual ones. If a bullet conflicts with
the current prompt, attachments, or source documents, the current case
wins. Use bullets to choose checks, frameworks, conventions, output
structure, and failure guards; verify the result independently.
